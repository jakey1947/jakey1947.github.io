Stygian Descent

A Sidescrolling Roguelike created in a team of 6 as my capstone project for my Bachelors degree. I was the lead programmer on the project spearheading the development of gameplay features and tooling for the game designers. We hit quite a few road bumps throughout the project, we overestimated the speed that our team could work at which brought along some rescoping, as well as some unforseen circumstances which interupted our work for weeks at a time. Eventually we settled for hand crafting levels rather than full procedural generation, no boss fights and only a few enemies to make sure we could hit the goals for release time.

Procedural Content

The levels in Stygian Descent are hand crafted, with procedural geometry generated from a base mesh. The level designer can create the level by using blenders mesh modelling tools, then the model and asset placement for the platforms will be handled automatically and imported using BEngine Unity by Mifth.

I'd have loved to make a more complex level generator and we did have fully procedural levels in our maximum scope but we simply didn't have enough time to implement it.

Items and Inventory

The items system was built using Scriptable Objects and a Chain of Command structure for the effects. When an item is added to the inventory, stats would be reset, then each passive item would update them using either a multiplier or custom script.

Active items on the other hand would be triggered via events such as player hit, heal, attack, jump etc. When an event is called, the relevant stats such as output damage would be passed through each item one by one for them to make their changes before it's applied to the enemy or player.

Rarity and Crafting

Throughout the game, the player can gain items in a Vampire Survivors like draw while in combat, as well as by using crystals at a crafting station for rarer items.

Each item is placed into a bucket for rarity, when a draw is started, a bucket is selected on an exponential curve, then an item is selected from that bucket at random. This is done 3 times and the player can pick the item they like.

Enemy AI

Enemy AI was handled using State Machines with scriptable objects. this allowed AI to be built by sticking behaviours for roam, persue and attack together without extra code, this didn't turn out to be much of an advantage given the end scope though.
